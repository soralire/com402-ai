Experiment plan for cxl_numa_csma

Model:
  CPU workers -> CXL Switch queue -> Type-3 Memory Device -> NUMA remote memory backend

Fixed configuration:
  bin=./cxl_numa_csma
  duration=5
  mem_node=1
  cpu_node=0
  mem_mb=512
  touches_per_req=4096

Sweep configuration:
  modes=0 1 2
  loads=10 30 50 70 90
  seeds=1 2 3
  threads=8 16
  queue_depths=4 8
  device_workers=1 2
  total_runs=360

Interpretation:
  threads > queue_depth recreates CXL Switch contention after queue_depth was expanded.
  queue_depth controls global concurrent slots in the CXL Switch / Type-3 queue.
  device_workers controls Type-3 backend service parallelism.
  Compare retry rate, goodput, p99 latency, and avg_cwnd across modes at the same load.
